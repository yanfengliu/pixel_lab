## 2026-04-24 — Multi-reviewer review + fixes

**Action:** Dispatched Codex (`gpt-5.4`), Gemini (`gemini-3.1-pro-preview`),
Claude (`claude -p`), and an internal superpowers code-reviewer subagent
on the branch diff. Consolidated findings, fixed the high-impact ones,
re-ran the full suite and build.

**Reviewer comments by theme:**

*Design / module boundaries*
- Gemini, Claude, subagent all flagged `ui/*.tsx → app/store` as a
  one-way graph violation. **Resolved** by moving `store.ts` from
  `src/app/` to `src/ui/` (same layer as its consumers). Added
  drift-log row.
- Codex and subagent flagged PNG decoding living in `core/` while
  ARCHITECTURE had placed it under `io/`. **Resolved** by updating
  ARCHITECTURE: PNG codec is pure (no DOM) and is used by both
  `core/export` (atlas encode) and `io` (import decode), so it belongs
  in `core`.

*Correctness*
- Codex, Gemini, Claude, subagent: `Buffer.from(...)` in `core/png.ts`
  would throw `ReferenceError: Buffer is not defined` in the Vite
  production bundle. **Resolved** by importing `Buffer` from the
  `buffer` npm polyfill; verified `dist/assets/*.js` contains the
  shimmed Buffer.
- Codex: GIF disposal type 3 (restore-to-previous) was being treated as
  type 2 (clear region). **Resolved** with a pre-blit snapshot + restore;
  added a dedicated test.
- Codex, Claude, subagent: `loadProject` didn't rehydrate GIF sources —
  a saved `.pixellab.json` would open with empty GIF previews and fail
  at export. **Resolved** by decoding the preserved `imageBytes` with
  `decodeGif` on load.
- Codex: `buildManifest` and `buildExport` could collide on duplicate
  animation names or on sanitized-identical names. **Resolved** with a
  `Set`-based dedup in `buildManifest` (throws on duplicate) and
  uniqueness in `buildExport` via `__2`, `__3` suffixes, plus
  store-level unique-name enforcement on add/rename. Added tests.
- Claude: `detectKind` only checked 4-byte GIF prefix. **Resolved** by
  verifying the full GIF87a/GIF89a magic plus the 8-byte PNG signature.
- Claude: `Canvas.tsx` swallowed slicer errors silently. **Resolved** by
  surfacing to Shell via `onSliceError` and rendering a red banner.
- Claude: `AnimationsPanel.handleAddAll` created an animation as a side
  effect of `??`. **Resolved** by guarding against zero refs and
  creating-on-demand.
- Claude: `SlicerControls` number inputs accepted NaN / 0 and crashed
  the slicer. **Resolved** with a `readNum` clamp.
- Claude: `projectFromJson` had no schema validation. **Resolved** with
  a dedicated validator that throws descriptive errors.

*Security*
- Gemini, subagent: `TopBar` used `project.name` directly as a
  filename. **Resolved** with `sanitizeFilenameStem`.
- Gemini: `ids.ts` used `Math.random` in the non-UUID fallback.
  **Resolved** with `crypto.getRandomValues` as an intermediate step.

*Performance*
- Gemini, Claude: `sliceAuto` O(n^3) merge loop. **Resolved** with a
  single union-find over the bounding-box index list.
- Claude: `packFrames` had an O(n^2) `ordered.find` inside the blit
  loop. **Resolved** with a `Map`.

*Robustness*
- Claude: `persist.fallbackOpen` detached `<input>` silently failed on
  Firefox and never resolved on cancel. **Resolved** by mounting the
  input into `document.body` and wiring a `cancel` listener.
- Gemini: `serialize/base64` spread 32 KiB into `fromCharCode.apply`
  risking RangeError on older Safari. **Resolved** by chunking at 4 KiB.
- Gemini: `Canvas` manual drag leaves a stuck draft if mouseup lands
  outside the canvas. **Resolved** by listening on `window` for
  move/up while a draft is active.
- Subagent: `manifest/per-frame` emitted `durationMs: 0` when timing
  info was missing. **Resolved** with a 100ms default.

*Test coverage*
- All reviewers flagged missing tests for `drag-drop`, `png encode/decode`,
  `loadProject`, and duplicate-name export. **Resolved** by adding
  `test/io/drag-drop.test.ts`, `test/core/png.test.ts`,
  `test/integration/load-project.test.ts`,
  `test/core/manifest-collision.test.ts`,
  `test/core/export-collision.test.ts`. UI component rendering tests
  remain a gap (see below).

**Out-of-scope / deferred to v1.1 (documented, not fixed):**
- Row-grouping UI (split a grid-sliced sheet by row into named anims).
- Manual rect resize handles.
- Pixel grid overlay at ≥8x zoom.
- React Testing Library component tests (shell, canvas, preview bar).
- Visual pixel-diff fixtures (`test/fixtures/`).

**Result:** 74/74 tests pass, `npx tsc --noEmit` clean, `npx vite build`
successfully bundles with `buffer` polyfill (135 kB gz).

**Reasoning:** Addressed reviewer findings in priority order: anything
that would crash in the browser (Buffer), corrupt output
(duplicate-name), or break round-trip (GIF reload) was fixed now;
cosmetic / stylistic items and speculative test coverage were deferred.
The spec v1 is satisfied; v1.1 items are logged here rather than as
