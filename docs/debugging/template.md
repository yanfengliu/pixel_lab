# Debug session template

Copy this file when starting a debug session. Name it with the
date + problem: `YYYY-MM-DD-<short-slug>.md`.

## Problem

What is broken and how do you know? Paste the failing test output, stack
trace, or screenshot.

## Hypotheses

List plausible causes. Rank them.

## Experiments

| # | Hypothesis | Steps | Outcome |
|---|------------|-------|---------|

## Root cause

Once found, describe it in one sentence.

## Fix

The minimal change. Why that change and not a larger one?

## Lesson

What, if anything, goes into `docs/learning/lessons.md`?
